# Resource Allocation in Emergencies: A Multi-Agent Learning Approach

## Problem Statement

City law enforcement agencies face:
- Limited police resources with dynamic, unpredictable criminal activity
- Need for rapid response to time-critical situations
- Inefficient manual allocation that cannot adapt in real time
- Complex coordination required among multiple police units

Traditional methods (greedy nearest-agent, random allocation, static zones) fail to optimise
resource utilisation in dynamic environments.

---

## Our Solution

A MARL system where agents learn through interaction on a planar graph:

| Component | Detail |
|-----------|--------|
| Environment | 19-node planar graph (Voronoi-based city layout) |
| Police agents | 5 — implicitly coordinate to locate and catch the thieves |
| Thief agents | 2 — learn to evade capture |
| Algorithm | Independent Q-Learning (one Q-table per agent) |
| Evaluation metrics | Catch rate, average reward, steps to first catch |

---

## Results

### Performance Comparison (100 evaluation episodes)

| Method | Avg Reward | Avg Steps to Catch | Catch Rate |
|--------|-----------|-------------------|------------|
| **MARL (Ours)** | **-11.57** | **2.36** | **100.0%** |
| Greedy | -12.65 | 2.36 | 100.0% |
| Random | -11.88 | 2.46 | 100.0% |
| Static | -182.49 | 1.30 | 86.0% |

### Key Takeaway
- MARL, Greedy, and Random all achieve **100% catch rate** — the graph is small enough (19 nodes, diameter 4) that all three reliably catch both thieves
- **Static fails** — 14% escape rate and massively negative reward (-182.49) confirms that movement is essential
- MARL achieves the **highest reward** among moving policies, meaning it wastes fewer steps than Greedy
- Results are reproducible — see `results/final_report.txt`

### Convergence
- Training Episodes: 2,000
- Final epsilon: 0.010 (near-greedy policy)
- alpha = 0.1, gamma = 0.9

## Methodology

### Environment

**Graph:** 19-node planar graph generated with Voronoi diagrams (seed=42). Weighted edges represent travel distances between nodes. Graph stats: 19 nodes, 47 edges, avg degree 4.95, diameter 4.

**Agents:** 5 police agents + 2 thief agents, all operating simultaneously in the same environment.

**State space:** Positions of all active police agents and all uncaught thieves, encoded as node indices.

**Action space:** `Discrete(max_degree + 1)` — move to any neighbouring node, or stay. Valid action count varies per node (min 4, max 8 on this graph).

**Reward structure:**

```
Police catches thief     : +200 (police),  -200 (thief)
Thief evades each step   : +1   (thief)
Move to neighbour        : -1   (all agents)
Stay in place            : -0.5 (all agents)
Thief escapes (timeout)  : -50 per uncaught thief (police)
```

### Algorithm — Independent Q-Learning

Each agent maintains its own Q-table and updates independently without explicit communication:

```
Q(s, a) ← Q(s, a) + α [ r + γ · max Q(s', a') − Q(s, a) ]
```

| Hyperparameter | Value |
|---------------|-------|
| Learning rate (α) | 0.1 |
| Discount factor (γ) | 0.9 |
| Initial epsilon | 0.30 |
| Epsilon decay | 0.995 per episode |
| Final epsilon | 0.01 |

Police agents develop emergent coordination — they learn to spread across the graph and corner thieves without any explicit communication protocol.

### Baselines

| Method | Description |
|--------|-------------|
| Greedy | Each police moves one step toward the nearest thief (shortest path) |
| Random | All agents choose actions uniformly at random |
| Static | All agents stay in place (no movement) |

## Key Findings

1. **MARL converges** — police agents learn effective pursuit strategies over ~500–1,000 episodes.
2. **Emergent coordination** — police implicitly spread out and corner thieves rather than clustering together.
3. **Outperforms baselines** — higher reward and catch rate than Greedy, Random, and Static methods.
4. **Scalable framework** — supports larger graphs and more agents with no code changes.

---

## Limitations

- Tabular Q-learning does not scale well beyond small graphs — the state space grows as O(N^(police+thieves))
- Agents have full observability of all other agents' positions
- No explicit communication between agents

---

## Future Work

1. Scale to larger city graphs (100–1,000 nodes) using Deep Q-Networks (DQN)
2. Partial observability — agents see only a local neighbourhood
3. Explicit communication protocols between police agents
4. Hierarchical learning (dispatch strategy + low-level movement)
5. Validation on real-world city crime/emergency datasets
